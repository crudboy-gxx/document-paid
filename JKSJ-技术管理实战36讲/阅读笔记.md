# 自我倾听
## 1工程师职业发展方向
1. 工程师的职业发展大类和细分方向。
+ 技术：架构师（广度、技术整体性、业务解决方案）、科学家（专项技术深度、精细度）
+ 管理：技术管理者、职业经理人
+ 创业：创始人（行业发展、商业、经营）、技术合伙人/技术高管
+ 顾问：投资人、管理顾问
2. 管理技能对于后续发展的重要性。
无论走哪个方向，规划、带人、沟通、执行等管理能力都是通用的。对于技术人来说，无论是否做管理岗位，所有的职业发展都会围绕着技术和管理这两条腿走路。

## 2寻找管理的内驱力
1. 选择走管理职业路线的外部因素。
不得已的选择、别人眼里的成功、不辜负组织的期待、对做管理的主管遐想。 
2. 如何判断自己是否存在做管理的内驱力。
是否认同管理的价值。是否对管理工作很期待享受。是否看中在管理方面的成长。     

## 3走向管理岗位的三方面因素
1. 天时、地利、人和三方面阐述走向管理岗位。
+ 天时方面，机会、大背景。管理需要长期的实践积累，去能积累的公司做管理是合适的选择。
+ 地利方面，自身优势、能力、工作内容。负责最全局的模块和最核心的技术模块有利于走向管理。
+ 人和方面，能否得到他人的支持。需要公司给与机会、上级给与指导和方向、平级给与陪伴和支持、家人给予情感。
2. 缺失天地人的情况，在工作中关注哪些方面对走向管理更有帮助。
+ 关注项目流程如何优化，合作机制如何建立，新员工如何培养，团队氛围如何建设等管理事务。

## 4对管理的患得和对技术的患失
1. 管理与技术无法兼顾导致内心冲突的因素。
+ 转管理之前没有仔细了解过管理。面对全新的事务，因为无法掌控而感到恐慌或焦虑。
+ 刚开始做管理，管理无法让自己安心，无法依靠管理安身立命。
+ 认为技术才是自己的大本营。以往依靠技术带来的成功，让自己非常依赖技术。
2. 做技术管理并不是放弃技术，不用患失。
做技术管理后放弃了编码，并不是放弃了技术，而是对待技术的视角不同，以及学习技术的方式有所变化。
+ 从目标和整体结果来看待技术。为了达到目标，我会把技术作为一个手段来看待，而不是做技术的时候把技术做好。我需要借助每个人的技术能力去做更多更大的事情。
+ 一定得有更高效的学习方式，掌握更多的技术，并快速判断如何搭配使用。一种方式是通过建立组织内的学习和共创机制，借助团队的力量来提升技术判断力。另一种方式是通过请教专家，专家一般能给出高屋建瓴、醍醐灌顶的认知。这两种方式，比读书和写代码更高效。
3. 做管理能锻炼自身能力，不用患得。
+ 做管理对个人成长和个人发展来说不会失败。做管理是一种修改，只要持续不断的学习、实践，造诣和能力就越来越高。
+ 管理做不好，并不是没有回头路。刚转管理的工程师，距离技术还是很近，回头做工程师没有门槛。
+ 做管理积累的经验能力，完全可以迁移到技术带头人上。在做管理时锻炼的全局视野、规划能力、结果导向意识、项目管理方法、协调沟通能力等，在成为高级工程师或尝试新方向上都会发挥很大作用。
4. 认清现实
+ 编码时间的减少，是不可避免的。现实是即便回头做技术，也不再是听指挥的一线工程师。要么做技术管理，用更高的视角来看待技术；要么继续做工程师，也要用更高的视角看待技术。
+ 无论做技术管理，还是做技术架构师，开启技术“升维”之路，是在在所难免的。工作视角要从以下方面进行升级：从目标出发看待技术，从评估的角度去看待技术，从借助自己的技术到借助大家的技术。

## 5如何保持技术判断力
1. 想对技术进行判断，就需要做技术评估，所以，通常我们所说的技术判断力就是技术评估能力。技术评估的三个重要维度：结果评估、可行性评估、风险评估。
+ 结果评估，希望得到什么结果，要不要做。从哪几个维度去衡量结果，从哪几个技术指标去验收结果。
+ 可行性评估，能不能做，值不值得做。值不值得做是考虑成本收益问题，而成本就有很多方面要考虑。
  + 资源投入成本，投入的人财物时。投入的资源是否与产出的相应的价值。
  + 维护成本。维护成本的来源有四方面，技术选型、技术升级、问题排查、代码维护。
    + 技术选型成本，在做“技术选型”时，选择不成熟技术所带来的成本，选择新技术的成本和风险比较大。
    + 技术升级成本，在“技术设计”时，兼容性和扩展性带来的后期升级的难度和成本。
    + 问题排查成本，在“技术实现”时，后期上线后系统运行过程中产生问题排查的时间成本。
    + 代码维护成本，在“编写代码”时，代码缺失情况下，在升级系统理解逻辑产生的时间成本。
  + 机会成本。把资源花费到这件事上，等同于放弃了另外一件事，放弃的这件事将来会产生什么影响。
  + 协作成本。一个方案的协作方越多，需要沟通协调的成本也越高，也就越不可控。
+ 风险评估。风险评估通常考虑几个方面，哪些技术风险需要未雨绸缪，哪些情况下会带来损失和最大损失的边界是什么。
2. 新经理的技术判断能力，通常来自于之前对技术的实操带来的经验。做管理之后，技术评估的要求更高，技术研究的时间却更少。那么，提升技术判断力方式也需要转变，需要去依靠团队和更广的人脉，去拓展技术视野和技术判断力，并且这种从周围人身上汲取知识将作为获取知识的主要方式。常用的方式可以采用：建立技术学习机制、专项技术调研项目化、和技术大牛交流、听取工作汇报。
+ 建立技术学习机制。盘点业务对技术的需求，成立与核心技术小组，让团队各个方向保持技术敏感度，要求小组定期交流和分享，以此保持自己的技术敏感度。
+ 专项技术调研项目化。如果某项技术对业务有重要价值，专门立项做技术调研，并要求负责人做调研汇报。
+ 和技术大牛交流。越是厉害的人，越能深入浅出的把技术讲明白，与他们交流，是很好的学习途径。
+ 听取工作汇报。阅读员工的周报、季报等汇报，沟通讨论，相互切磋，也是一种很多学习方式。

## 6常用管理风格和各种风格适合的场景
1. 管理风格的本质是管理者和团队的协作方式，也就是管理者和团队的位置关系，即管理者站到团队的什么位置。位置关系大体分四类：发号施令型、以身作则型、激发辅导型、无为而治型。
+ 指令式管理：目标和结果、指挥、下级执行力。
+ 支持式管理：带有冲锋、分担工作、小团队、攻坚时刻、提升凝聚力时。
+ 教练式管理：全局和方向、辅导和启发、提升团队、建立梯队。
+ 授权式管理：目标和结果。
2. 不同的风格，在不同的场景下，会有不同的适应程度。例如，团队刚起步，人比较少，适合支持式；后续人多了，需要固化流程和制度，适合指令式；等流程固化，需要引导大家提升，不断优化流程时，适合教练式；项目比较城市且有合适负责人，防守全权负责，适合授权式。
3. 自我组织、自我管理也是一种管理方式。研发人员都是逻辑性和执行力都比较强人，有明确的目标，给与足够的支持和资源，让大家对结果负责，实现自我管理，并充分锻炼自己的能力。伴随着自我的管理，自我目标和价值的实现，组织的目标和价值自然也被实现，这是一种基于信任的管理。

## 7如何克服不自信
1. 新经理不自信的原因主要有如下几点：管理经验不足和能力欠缺，和团队成员对立比较，背负着沉重的包袱。
+ 管理经验不足和能力欠缺（自我能力）。能力三核知识、能力、才干。建立能力分层意识，迁移技能和才干，学习掌握管理知识。
+ 和团队成员对立比较（角色认知）。将自己与团队成员比较中抽离，看想远方，考虑如何让大家发挥才智，达成共同目标。不是管束和控制，而是引导和支持大家。
+ 背负着沉重的包袱（自我认同）。首先建立沟通机制和反馈通道。询问上级和合作伙伴，自己哪些方面做的比较好，清楚自己擅长的工作方式。

# 角色认知
## 8管理都做哪些事儿
1. 管理框架、管理地图、管理图谱目前没有。 工业时代的外驱力，互联网时代的内驱力。管理的复杂性随着时代和社会环境改变而改变。
2. 管理有两个主要要素，一个要素是人和组织，另一个要素是事务。
3. 管理工作五个要素：角色认知、管理规划、团队建设、任务管理、管理沟通
+ 角色认知：
+ 管理规划：看方面。职能、目标、团队、路径
+ 团队建设：带人。激励、分工、协作、梯队、文化、能力
+ 任务管理：做事。有效执行、轻重缓急、流程机制
+ 管理沟通：

## 9角色认知
1. 一个人的行为、能力、价值观，都源于一个最根本的认知，这就是角色定位。
2. 从工程师到管理者角色发生了哪些转变。
+ 1）工作职责：完成手工工作；达成团队目标。
+ 2）负责对象：自己；上级和下级。
+ 3）关注焦点：过程导向，脚下的路；目标和结果导向，远方的目标。
+ 4）工作内容和能力要求：个人贡献者；看方向、带人、做事更多维度和立体能力。
+ 5）任务来源：上级安排；自己筹划和上级沟通确认。
+ 6）实施手段：个人；团队。
+ 7）合作维度：平级；上、下、平360度。
+ 8）合作关系：竞合；合作。
+ 9）思维方式：执行思维，过程、细节、风险、成本；规划思维，收益。
+ 10）技术视角：技术实施角度；技术评估判断角度。

## 10几种不合适的管理风格
1. 过程导向、被动执行。
2. 大包大揽、唯我最强。
3. 带头大哥、当家保姆。
4. 单一视角、固话思维。
5. 自扫门前雪，固守边界。
6. 患得患失。

# 管理规划
## 11切入点
1. 要从问题驱动型思维向规划驱动型思维转变，规划团队未来的方向。
2. 职能（你团队是干什么的）、目标（你团队想做出什么成果）、团队（你依靠什么样的团队）、路径（你需要投入哪些资源）。

## 12如何界定团队是干什么的
1. 基本的职责和升华的使命。职责是团队的下限，是解决团队生存问题，使命是团队的上限，是解决团队幸福问题。
2. 设定团队职责和使命的方法和步骤
+ 收集信息：向上沟通，初始定位，评估维度；向下沟通，对未来发展的期待；左看右看，与其他部门衔接，不覆盖兄弟部门职责；你的理解，团队的使命和愿景往往来自你的设想。
+ 提炼和升华：职责的提炼，使命的升华，确定衡量维度。
+ 确认和主张

## 13如何设定合理的团队目标
1. 目标对于团队意味着什么：我和上级的诉求；资源的有效配置；执行力；凝聚力；激励。
2. 目标设定符合SMART原则，目标要是明确的、可衡量的、可到达的、相关的、时限的。
+ 目标设定要有挑战性且努力能到达的。目标要做的激进些。
+ 目标的描述有两类。可量化的指标和不可量化的指标，不可量化的指标用关键结果和实现效果衡量。
3. 目标设定中常见的问题。
+ 基于现有资源做目标，而不是基于远方的目标往前推进。
+ 目标不明确。不是因为目标太大导致，而是因为过程化描述引发目标不明确。面对这个问题使用结果导向的描述。
+ 目标没有刻意向团队成员传达。
+ 目标总是被迫变来变去。面对这个问题设定专业目标。
4. 专业目标的设定。专业目标对应的是业务目标，业务目标是上级对业绩的要求，专业目标是我和团队制定，属于自我要求。第一，选择要提升的关键维度；第二，设定目标。

## 14如何来规划团队的组织结构
1. 根据团队目标的设定去梳理团队。
+ 此目标不是业务目标，而是你希望在某个时间节点到来的时候，把团队发展成什么状态。
+ 从团队规模、分工、梯队，这三个要素来描述。
2. 从资源角度来审视团队。
+ 作为管理者，在盘点当前人力和预算人力的时候，需要有成本意识。
+ 如何做人力预算：取决于你对业务的理解，以及希望达成的目标。另外，可以参照行业资源配比情况。
3. 从人才培养角度来看梯队建设。
+ 到下一个时间节点，你需要重点培养哪些人，给他们什么样的平台和空间，你能够提供什么样的指导和支持，期待他们能够胜什么职能和角色。
+ 如何估算团队消化能力。看看团队中谁能带人，带几个人合理。建立健全额新人培养机制。
4. 人才规划报告：规模，当前人数、预算人数、总人数；分工，业务方向划分、业务线负责人；梯队，人员级别分布、人员结构梯队。
   
## 15我要申请哪些资源
1. 资源丰富性。除了人财物这些常规的资源，还有几类容易忽略的资源，分别是时间、信息、权限。评估一个平台是否有发挥空间时，不能只看职位和人员，还要看能否得到全面的支持。
2. 实现路径多样性。
+ 手段：自研、招聘、借调、合作、外包、云服务、买方案。
+ 维度：时间成本、资金成本、维护成本、合作成本、替换成本。
3. 人才持续性。往往太高估招聘能解决的问题，太低估人才选育的成本。招聘作为一种迟缓解决问题的手段，更多的是看长线是否有需求。
